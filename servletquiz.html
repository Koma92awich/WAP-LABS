Solutions to Introduction to servlet Quiz

1. Web server refers to an execution infrastructure that handles HTTP requests and responses. A web container container refers to a component of the web server that interacts with Java servlets and manages the lifecycle for servlets.

2.A servlet is a Java programming language class that is used to extend the capabilities of servers that host applications accessed by means of a request-response programming model. Mosttly to extend application hosted by web servers.

3. By using the Hyper text transfer protocol over TCP/IP. When a request comes to the web server, if the server sees the request is for a servlet, it passes the request data to the servlet container. The servlet container locates the servlet, creates request and response objects and passes them to the servlet, and returns to the web server the response stream that the servlet produces

4.Request objects are created by the servlet/web container.

5. States in the Life cycle of a servlet
-Servlet class is loaded.
-Servlet instance is created.
-init method is invoked.
-service method is invoked.
-destroy method is invoked.

6.The web/servlet container calls the init method only once after creating the servlet instance. The init method is used to initialize the servlet. It is the life cycle method of the javax.servlet.Servlet interface. Syntax of the init method is : public void init(ServletConfig config) throws ServletException {.........} .

7. You should override the doGet method from the HttpServlet class.

8. Servlets are multithreaded in the sense that each time the service method is invoked , for each request there is a seperate thread running.

9. Servlet instance variables are shared by all threads created for each request when the service method is invoked. Hence not thread safe.
